Method

First, we accumulated data from online scholarly articles and books from our library. We searched for maps, movements of diseases and facts of disease spread. Later, we emailed tribal leaders for any information and guidance they could provide.

Next, we compiled these datapoints into an Excel sheet and added metadata to help with the graphing process. We noted the date, location, disease, source, and any related information that could either explain or be explained from this event.

While searching for data, we realized that statistics about Native American infection rates were sparse compared to European colonizers. For example, we found more information about syphilis than other diseases, likely because it affected European colonizers.  As we continue the project, we hope to be able to add more disease spreads but finding data will always remain an obstacle. Records were not kept in writing by Native Americans and reaching out to tribal leaders has not yielded a response yet, so we will need to keep finding sources.

At the same time, we built a website first through WordPress, but eventually we changed it to GitHub, using Publii to design it. We looked to other map projects such as Photogrammar and al Turraya to learn what features were possible in Leaflet (our mapping tool), and which ones we wanted to add to our own map. We listed features to incorporate, including multiple map backgrounds, a time slider and connected points on the map to show spread.

As we gained new insights, we wrote them down into a PowerPoint and document. We presented what knowledge we gained to our peers. With their help, we created a document with feedback and ways for us to improve and clarify our project.

The map came next, with us adding datapoints to create a basic map, and adding features atop. Unfortunately, we have not added all the features yet, but we will continue to add and improve our map and embed it into our website.  

Finally, we compiled our information onto our website, and began to review our project. In the future, we will continue to look for datapoints on multiple diseases and add new tabs for each disease. Hopefully we will have multiple disease progressions and details on each one to help those interested in our topic.

 

The programs we used include Microsoft Word, Google Drive, Sheets, Docs, Publii, Github, Leaflet and an internet browser. These tools were chosen to help us create our project in an efficient and accurate way. Microsoft Word and Google Docs were used to write, with Word being superior offline and Docs being superior for collaboration. Google Sheets allowed each of us to add datapoints gathered to a common place. These datapoints allowed us to build the map in Leaflet, which we chose for its features (gathered in other DH projects) and compatibility. Github and Publii were chosen to build and host the website.
